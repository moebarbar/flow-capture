Create a complete, enterprise-grade SaaS platform for automated workflow documentation, step-by-step guide creation, interactive training materials, and video generation. This is a comprehensive full-stack application with Chrome extension, web application, video generation engine, team collaboration, analytics, and full admin panel.

PROJECT OVERVIEW

## WHAT TO BUILD
A platform similar to Tango.ai that allows users to:
- Automatically capture workflows by recording browser interactions via Chrome extension
- Generate step-by-step guides with screenshots and AI-generated descriptions
- Create interactive walkthroughs that guide users through processes in real-time
- Convert guides into engaging explainer videos with AI voiceover and animations
- Generate direct links to specific steps and elements within guides
- Collaborate with teams in shared workspaces with real-time co-editing
- Manage everything through a comprehensive super admin panel
- Embed guides anywhere (websites, knowledge bases, apps)
- Track detailed analytics and measure guide effectiveness
- Full subscription management with multiple pricing tiers
- Complete branding customization through admin panel
- Manage landing page content dynamically from admin panel

## TECHNOLOGY STACK

Frontend:
- React 18+ with TypeScript
- Tailwind CSS for styling
- Framer Motion for animations
- Lucide React for icons
- Recharts for analytics charts
- React Router v6 for navigation
- Zustand for state management
- React Query for server state

Backend:
- Node.js with Express.js
- PostgreSQL for primary database
- Redis for caching and sessions
- AWS S3 or CloudFlare R2 for file storage
- JWT for authentication
- Socket.io for real-time features
- Bull for job queues
- Stripe for payment processing
- SendGrid for emails

Chrome Extension:
- Manifest V3
- Vanilla JavaScript for performance
- Chrome APIs for screenshots and tab management
- Communication with backend via REST API

Video Generation:
- FFmpeg for video processing
- Canvas API for animations
- Text-to-speech API (Google Cloud TTS or AWS Polly)
- Custom rendering engine

## EXTENSION STRUCTURE
Create a Chrome extension with these files:
- manifest.json (Manifest V3 configuration)
- popup.html/js/css (main popup interface)
- content.js (injected into all web pages to capture interactions)
- background.js (service worker managing state and communication)
- recorder.js (intelligent workflow capture logic)
- highlighter.js (visual element highlighting)
- options.html/js/css (extension settings page)

- Manifest version 3
- Extension name: "WorkflowCapture Pro"
- Permissions: tabs, activeTab, storage, scripting, webNavigation, desktopCapture, contextMenus, cookies
- Host permissions: <all_urls>
- Background service worker
- Content scripts injected into all pages
- Browser action with popup
- Extension icons in 16x16, 48x48, 128x128 sizes

## INTELLIGENT WORKFLOW CAPTURE

Content Script Features:
- Capture all user interactions:
  - Click events on buttons, links, icons, images
  - Form input changes (text, dropdowns, checkboxes, radio buttons, file uploads, date pickers)
  - Form submissions
  - Navigation events (URL changes, back/forward, page refreshes)
  - Tab/modal switches
  - Accordion expansions
  - Keyboard shortcuts
  - Drag and drop actions
  - Hover-triggered interactions
  - Scroll events (debounced)
  - Focus events for accessibility
- For each interaction, capture:
  - Element information (tag name, ID, classes, text, label, placeholder, type, name, ARIA attributes, role)
  - CSS selector (intelligent, unique selector generation)
  - XPath (alternative selector method)
  - Element position (bounding rectangle)
  - Visibility status
  - Relevant computed styles
  - Parent context
- Smart features:
  - Ignore recording UI interactions
  - Debounce rapid repeated actions
  - Group related actions
  - Detect form completion
  - Identify important UI elements automatically
  - Handle dynamic content (SPAs, AJAX)
  - Detect iframe interactions
  - Monitor network requests

## SCREENSHOT TECHNOLOGY

- High-resolution capture (up to 1920x1080)

  - Red border around clicked element (3px, rounded corners)
  - Subtle shadow for emphasis
  - Pulse animation on capture
- Smart cropping to focus on interaction area
- Full-page scrolling screenshots when needed
- Viewport annotations showing interaction context
- Optional mouse cursor position marking
- Click animation overlay
- Element information overlay (optional)
- Respect privacy mode (auto-blur detected sensitive fields)
- Compress images before upload
- Multiple quality settings

## DEEP LINK GENERATION

For each captured step, generate:
- Full URL with anchor to specific element
- If element has ID, use #elementId as anchor
- If no ID, generate hash-based anchor from CSS selector
- Store mapping of anchor to element selector and XPath
- Detect if element is clickable (button, link, etc.)
- Create clickable hotspot data with:
  - X, Y position (percentage relative to screenshot)
  - Width and height (percentage)
  - Action type (click, navigate, etc.)
  - Target URL or action
- Validate URLs and test accessibility
- Handle dynamic URLs with query parameters
- Create fallback selectors if primary fails

## NATURAL LANGUAGE DESCRIPTIONS

Auto-generate smart descriptions for each action:
- Button clicks: "Clicked '[button text]' button"
- Link clicks: "Clicked '[link text]' link"
- Input fields: "Entered text in [field label] field"
- Dropdowns: "Selected '[option]' from [dropdown label]"
- Checkboxes: "Selected/Deselected [checkbox label]"
- Radio buttons: "Selected [option] option"
- Form submissions: "Submitted [form name] form"
- Navigation: "Navigated to [page title]"
- Use context clues from ARIA labels, placeholders, surrounding text
- Vary action verbs naturally (clicked, selected, entered, navigated, opened, closed, expanded, collapsed)
- Create full sentences that read naturally

- Retry failed uploads automatically

- Update extension badge with step count during recording
